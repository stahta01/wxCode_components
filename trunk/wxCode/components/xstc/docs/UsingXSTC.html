Getting Started

Create an XSTC instance.

XSTC* MyXSTC = new XSTC(this, wxID_ANY, wxDefaultPosition, wxDefaultSize, 0, wxT("MyXSTC"));

Give XSTC a config object if you want to use coloring from a config databse.

wxFileConfig* conf = new wxFileConfig(wxT("test"), wxT("NZ"), wxT("test.conf"), wxT(""), wxCONFIG_USE_LOCAL_FILE | wxCONFIG_USE_RELATIVE_PATH, wxConvUTF8);
MyXSTC->SetColorConf(conf);

Give XSTC a color database if you are using color names in the config database.

wxColourDatabase db;
MyXSTC->SetColorDbase(db);

If you want to use config based properties with the extension to lexer mapping functionality:
MyXSTC->useprops = true;

This bool will invert the fg and bg colors of the default marker if set to true. Defaults to false.
MyXSTC->markoutline=true;

Keywords are not set automatically by default, to enable that:
MyXSTC->usekeys = true;

This function initializes everything to default. It sets up folding and everything. It is called in the constructor.

MyXSTC->ResetStyle();

Also in order to view colors call a theme function. ResetStyle() does the opposite of one of these. Aslo a config based theme can be loaded in place of one of the built in themes.

MyXSTC->DarkStyle(); or MyXSTC->ConfigStyle(wxT("fire"));

Initially folding sets the default marker for all fold fields in the folding margin. In order to see a useful fold margin, call a folding function.

MyXSTC->FoldArrow(true); //with arrow and simple a bool argument will show or hide connecting lines, box and circle use them always.

To tell XSTC to load a file for keywords rather than pass the string directly to Scintilla, prepend "^ISFILE^"
MyXSTC->Keys_Set.cpp1 = "^ISFILE^" + fileobj->GetPath();
When loading from file multiple filenames can be used, the string is coma delimited. A coma between "^ISFILE^" and the first filename is optional.
MyXSTC->MatrixStyle();
MyXSTC->SetLexerX(MyXSTC->GetLexer());

In order to use some of XSTC's extra functionality there are three file related functions.
  1. LoadFileX()
  2. SaveFileX()
  3. CloseFile()
Scintilla doesn't actually load a file itself, wxScintilla makes it seem that way by loading a file and giving the contents to Scintilla. Because of this there is no need to actually "close" the stream or anything. The Close function is a shortcut for reseting everyting to zero basically. LoadFileX(), can call SetLexerX() and AutoExt() if you tell it to. By using this function most of the style color settings will be taken care of for you. The lexer can be choosen automatically by extension and if you have the option activeted keywords will even be set. SaveFileX() will call a few utility functions if you have those options activated. This way they can be run on the document automatically before it gets saved to disk.

If you don't want to use some functionality check config.h for information on build macros that will turn some of it off. If the coloring management isn't your style then you might want to just look at the bits that have funtionality you do like and impliment the rest your own way, as that is the majority of the codebase. XSTC does not add any functionality over wxScintilla it just makes things faster and easier to use.
Take a look at it and see what you think. I hope that you will like it. If you decide that you don't want to use it later just replace it with wxScintilla and go. You do not have to use the loadx, savex and setlexerx functions and you can always decide when to call coloring functions on your own. I hope that XSTC if flexible enough to be of use to most people looking for this type of solution.

XSTC/EXT/

Each file extension can be entered like this. The only required option is "LEXER". For extensions that will use the same properties, setting the "LANGUAGE" option can make things easier.
[XSTC/EXT/CPP]
LEXER=CPP #This can also be the number set in the header file.
LANGUAGE=C

These fields can also be put in the extension group, just ommit the "LANGUAGE" entry.
[XSTC/EXT/LANG/C]
PROPCOUNT=3 #only the number of properties set here will be read.
PROP1=fold.comment_1
PROP2=fold.preprocessor_1
PROP3=fold.compact_0

XSTC Functionality

  • Space->tab,tab->space
  • Trim trailing spaces
  • Numbered bookmarks 0-9
  • Bookmark margin is already to go with a marker
  • Brace matching
  • Margin type,size,order rearrangable
  • Lexer mapping by extension (in config)
  • Per language coloring(in config)
  • Folding is ready to go
  • Predefined keywords and optional auto keyword setup for languages, not all languages have keywords. The predefined keywords can be turned off with a macro.
  • Support for either wxStyledTextCtrl or wxScintilla, your app can as well use the macros, but they are not necessary either.
  • The default mouse bookmark can be set to outline or solid (largely depends on specific coloring choices)
  • Some macros for certain functions that are different between stc and wxsci, allowing you to make your program library portable.
  • Use of color database so you can use color names in the config (optional)
  • Dynamic color definitions in conf allow for different colors than the current theme.
  • Build macros turn off newer Scintilla options and any specific lexer
