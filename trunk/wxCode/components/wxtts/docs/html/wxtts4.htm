wxTTS bugs, problems and omissions

The following work is still needed or can be considered:

  • wxFreePhone needs further testing and debugging for memory problems in cleanup;
  • more error checking and reporting required;
  • (wx)FreePhone code could use rewriting for clarity, perhaps using a string class;
  • need control over how numbers, dates etc. are pronounced;
  • further samples would be useful, for example for reading large texts (which will need to break text into chunks and wait for a chunk to be read before reading the next one);
  • custom voices could be maintained in wxMbrola, with a mapping between voice name and voice parameters;
  • a wxTTS class could be created to unify wxFreePhone and wxMbrola;
  • some SAPI compatibility could be attempted;
  • need to produce standard dialogs that can be invoked to adjust parameters;
  • could have a second (customisable) dictionary, perhaps with a dialog to allow addition of words in a friendly manner (e.g. you have a selection of phonemes which you can put together, with a preview - maybe even a soundex-like algorithm to make an initial guess of what the phonemes should be);
  • need to write further makefiles, e.g. for Gnu-Win32, BC++;
  • need ports to other platforms.

See also wxFreePhone Discussion, wxMbrola Discussion.
